The Default Emitter

This page assumes that readers are already familiar with the emitter. If not then this page should be read before proceeding.

All SuperScript declarations (i.e., instances of DeclarationBase) reference an instance of emitter (by its Key property). However, when making a declaration, it is possible to omit an emitter Key.

In such cases, when not emitter Key is specified, SuperScript will instead reference the Key of the default emitter.

What is the default emitter?

The simple answer to this question is the emitter which will be used when no emitter is referenced.

So how does an emitter become the default emitter?

There are two ways in which an emitter may become the default emitter:

  1. Specifying IsDefault = true.

    When specifying an emitter in the config file, this is achieved through the isDefault attribute on the <emitter> element. 

                <superScript>
                <emitters>
                    <emitter key="[YOUR EMITTER KEY]" isDefault="true">
                    …
                    </emitter>
                &t;/emitters>
            </superScript>

    If multiple instances of emitter have IsDefault set to true, the first such emitter will be recognised as the default emitter.

  2. If no emitter has IsDefault set to true then the first emitter in the Emitters collection will be recognised as the default emitter.

It is possible to obtain the default emitter at any point in the project by calling 

    IEmitter SuperScript.Configuration.Settings.DefaultEmitter { get; }

The above read-only property can throw two exceptions which should be handled.

CollectionNotInstantiatedException

Thrown if the Emitters collection property has not been instantiated. This should be done for each HTTP request. See here for more information on initialisation.

NoEmittersConfiguredException

Thrown if no instances of IEmitter have been configured.
